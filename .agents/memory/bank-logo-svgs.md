## Storage pattern
SVG file contents are inlined as template-literal string constants in:
`artifacts/trustpoint-bank/constants/bank-svgs.ts`

Generated by Python script that reads each file and escapes backticks.
**Why:** Expo Metro bundler has no built-in SVG transformer; SvgXml requires a string, not a require().

## Rendering (BankLogo.tsx)
- GTBank, UBA: hand-coded JSX SVG (clean icon shapes, no text) — keep as-is
- TrustPoint: transparent PNG (icon_transparent.png) on transparent bg
- Opay: PNG on teal circle
- Access Bank, Moniepoint, First Bank, PremiumTrust, Stanbic IBTC: `SvgXml` on white pill
  - `size` prop = HEIGHT; width = `min(size × aspect_ratio, size × 3.5)`
  - White background + subtle border for wordmark pills
  - All others (no real logo): brand-color circle with initials fallback

## hasLogo field in banks.ts
`hasLogo: true` means the bank has a real graphical logo (either SVG from zip or coded icon).
Used to filter bank selection lists (register, account switcher add-bank) and transaction icons.
Banks with only initials fallback are `hasLogo: false` and excluded from selection UIs.

**Why:** "Only show banks with active logos" — user's explicit requirement.
